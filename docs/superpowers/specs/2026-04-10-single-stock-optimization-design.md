# 单股分析模块优化设计

*日期: 2026-04-10*

## 背景与动机

LinDangAgent 现有单股分析采用"四野指挥部"架构：3 位 Sonnet 将领并行分析 → 韩先楚否决 → Bull vs Bear 辩论 → Opus 林彪裁决，共 5 个 Phase、4-7 次 AI 调用、耗时 2-4 分钟。

核心痛点：

1. **评分一致性差**：同一只股票重复分析评分波动 10-15 分，三将领分歧大，区分度不足（大量股票堆积在 55-75 分）
2. **风格错位**：系统偏保守（基本面导向），用户是价值投机短中线作手，需要更重预期差/资金面/技术面
3. **信息噪声**：报告冗长但关键信号密度低，洋洋洒洒很多内容但决策价值不高
4. **缺乏自进化**：评分框架和权重写死，无法从实战结果中学习

## 设计目标

- 评分一致性显著提升（相同事实 → 相同分数区间）
- 注入用户的价值投机风格
- 精简报告，提高信号密度
- 建立从实战结果到 prompt/权重自我进化的闭环
- 所有影响评估质量的变更必须汇报用户确认

## 新架构总览

### 从 5 Phase 到 3 Phase

```
旧：侦察 → 三将领并行 → 审查替补 → 韩先楚否决 → 辩论 → 林彪裁决 → 组装
新：侦察 → Opus深度分析(Round 1) → Opus自我质疑(Round 2) → 组装
```

| 维度 | 旧架构 | 新架构 |
|------|--------|--------|
| AI 调用次数 | 4-7 次（3 Sonnet + 1 Opus + 重试） | 2 次（2 Opus） |
| 角色 | 3 将领 + 韩先楚否决 + 辩论 + 林彪 | 1 分析师（两轮） |
| 评分来源 | 三人评分中位数 → 林彪修正 | Round 1 出分 → Round 2 修正 |
| 容错层级 | 7 层（多角色互补） | 3 层（决策树锚定 + 自我质疑 + 代码校准） |
| 预计耗时 | 2-4 分钟 | 40-90 秒 |

### 三个核心模块

1. **分析引擎**（Phase 0-2）：Opus 两轮深度分析，内嵌价值投机风格
2. **评分框架**（贯穿）：决策树硬锚定，取代自由裁量
3. **进化引擎**（独立运行）：实战复盘 + 历史回测 + 权重自调节 + 变更汇报

### 设计依据

去掉多将领架构的理由：三将领看相同数据用不同风格打分，本身就是不一致的来源。学术研究表明同一模型的自我质疑（self-critique）比多 agent 辩论更能提升推理质量——同一个模型上下文完整，不存在信息损耗。

去掉林彪独立裁决环节的理由：在新架构下，分析师本身就是 Opus（最强模型），且已内置 Pre-mortem 和评分纪律。林彪原有的四项职能（独立初判、分歧裁决、Pre-mortem、评分校准）全部被新架构覆盖或代码化，作为独立环节的增量价值极小。

## 模块一：分析引擎

### Phase 0：侦察

保留现有情报采集逻辑（财报+资金+舆情+宏观+技术指标），增加：

- **注入进化经验**：从经验库中检索与当前股票相关的历史教训，拼入数据包末尾的 `【历史镜鉴】` 区块（检索机制见"经验检索"章节）

### Phase 1：Round 1 — 深度分析（Opus）

**风格定位**：价值投机短中线作手

**初始权重**（可被进化引擎调节）：

```
基本面: 10%（降权，体检式，只看硬伤）
预期差: 40%（提权，核心驱动）
资金面: 30%（确认信号）
技术面: 20%（辅助判断）
```

**输出结构（6 个必填区块）**：

| 区块 | 内容 | 约束 |
|------|------|------|
| 核心定调 | 一句话：做多/做空/观望 + 核心逻辑（≤20字） | 必须可证伪 |
| 催化地图 | 未来 30 天内的具体催化事件 + 预期时间窗口 | 每个催化必须有明确日期或条件触发点 |
| 四维评分 | 沿决策树打分，写出走过的路径 | 不允许自由裁量，必须对应决策树节点 |
| 资金确认 | 主力资金动向（北向/融资/大单）+ 量价关系判读 | 必须引用具体数字 |
| 技术研判 | 关键支撑/压力位 + 形态识别 + 趋势判断 | 必须给出具体价位 |
| 仓位建议 | 入场点/止损/目标位/持有周期 | 止损必须是具体价位 |

**关键改变**：
- 基本面压缩为"体检"（3-5 行，只看有无硬伤），删除冗长论文式分析
- 催化地图和资金确认提升为核心区块
- 每个区块有字数软上限，防止冗长

### Phase 2：Round 2 — 自我质疑（Opus）

把 Round 1 的完整输出喂回同一模型，切换为对抗性人格。

**强制任务**：

1. **找出 3 个可能导致亏损 20%+ 的致命理由**（禁止"大盘系统性风险"等廉价论据）
2. **逐条审视催化地图**：哪些是"已充分定价"的伪催化？
3. **检验资金面信号**：北向买入是被动指数调仓还是主动加仓？
4. **评分修正**：每个维度最多 ±10 分修正，必须说明修正理由
5. **Pre-mortem**：买入后 20 个交易日内暴跌 20%，最可能的 3 条路径，概率判定（高/中/低）

**硬约束**：
- Pre-mortem 有 ≥1 个"高概率"致命路径 → 综合加权上限 70
- 反驳未找到任何实质性问题 → 判定为"无效反驳"，代码层面检测反驳质量并要求重新质疑

### 最终评分合成（代码层）

Round 2 修正后的评分 → 木桶效应修正 → Pre-mortem cap → 输出最终分数

### 容错机制

| 层级 | 场景 | 处理 |
|------|------|------|
| L1 | Opus 调用失败 | 汇报用户，不静默降级 |
| L2 | Round 1 评分缺失 | Claude Sonnet 兜底重试 + 评分提取 prompt（保留现有三级重试） |
| L3 | Round 2 无效反驳 | 检测反驳质量，要求重做（最多 1 次重试） |
| L4 | 评分异常（全 0 或全 100） | 代码层钳位 + 标记异常 |

## 模块二：决策树评分框架

核心思路：相同的事实 → 走到相同的决策树叶子 → 相同的分数区间。消除"凭感觉打分"的随机性。

### 预期差维度（权重 40%）

```
Q1: 是否存在未被市场充分定价的催化事件？
├── 否 → 30-45分
└── 是 → Q2

Q2: 催化的确定性？
├── 已公告/已披露（A类证据）→ Q3
├── 可靠渠道预期（B类证据）→ Q3，上限-10
└── 纯逻辑推演（C类证据）→ 上限55分

Q3: 催化的时间窗口？
├── 30天内触发 → Q4
├── 30-90天 → Q4，-5分
└── >90天 → 上限60分

Q4: 市场定价程度？
├── 股价未反应（横盘/下跌中）→ Q5
├── 已部分反应（涨<10%）→ Q5，-10分
└── 已充分反应（涨>20%）→ 上限50分

Q5: 催化的量级？
├── 业绩拐点/行业变革/政策级 → 80-95分
├── 单季超预期/产品放量/订单落地 → 65-80分
└── 小事件/情绪催化 → 50-65分
```

### 资金面维度（权重 30%）

```
Q1: 主力资金方向？（北向+融资+大单净额，三者投票）
├── 三者同向流入 → Q2，基础70分
├── 两正一负 → Q2，基础55分
├── 两负一正 → Q2，基础40分
└── 三者同向流出 → 25-40分

Q2: 量价配合度？
├── 放量上涨/缩量回调（健康）→ +10~15分
├── 量价平稳 → +0分
└── 放量下跌/缩量上涨（背离）→ -10~15分

Q3: 筹码结构？
├── 机构近期加仓+集中度提升 → +5~10分
├── 无明显变化 → +0分
└── 机构减仓+筹码分散 → -5~10分
```

### 技术面维度（权重 20%）

```
Q1: 趋势状态？
├── 上升趋势（MA20>MA60，价在MA20上方）→ 基础65分，→Q2
├── 震荡整理 → 基础50分，→Q2
└── 下降趋势 → 基础35分，→Q2

Q2: 关键位置？
├── 突破重要压力位/平台 → +10~15分
├── 支撑位附近企稳 → +5~10分
├── 无明显关键位 → +0分
└── 跌破重要支撑位 → -10~15分

Q3: 形态信号？
├── 经典看多形态（W底/杯柄/旗形突破）→ +5~10分
├── 无明显形态 → +0分
└── 经典看空形态（头肩顶/M顶/跌破颈线）→ -5~10分
```

### 基本面维度（权重 10%）— 体检式

```
Q1: 是否存在硬伤？
├── 财务造假嫌疑/ST风险/巨额商誉减值 → 15-25分（熔断）
├── 业绩持续下滑>3季 → 30-45分
├── 无硬伤但平庸 → 50-60分
└── 无硬伤 → Q2

Q2: 业绩趋势？
├── 营收+利润双升且加速 → 75-90分
├── 稳定增长 → 60-75分
└── 增速放缓但仍为正 → 50-60分
```

### 综合加权修正规则（代码层）

| 规则 | 条件 | 修正 |
|------|------|------|
| 催化-资金共振 | 预期差≥75 且 资金面≥70 | 综合+3分 |
| 催化-资金背离 | 预期差≥75 但 资金面≤45 | 综合-5分 |
| 基本面熔断 | 基本面≤25 | 综合上限30分 |
| 木桶效应 | 任一维度≤30 | 综合上限60分 |
| Pre-mortem cap | Round 2 高概率致命路径≥1 | 综合上限70分 |

### Opus 微调空间

Opus 在走完决策树后可附加 ±5 分的"综合研判微调"，但必须明确写出微调理由，且幅度硬性限制在 ±5 分。

### 决策树进化

决策树不是写死的。节点阈值、分支结构、权重比例都可以被进化引擎调节（见模块三）。初始决策树是基于投研经验的合理起点，随实战数据积累持续优化。

## 模块三：进化引擎

### 三条进化通道

```
通道 A：按需批量复盘（用户触发）
通道 B：历史回测（夜间自动）
通道 C：权重与决策树调节（元优化，需用户审批）
```

### 通道 A：按需批量复盘

**触发方式**：用户通过 CLI 命令手动触发

```bash
python cli.py review                              # 复盘最近一周
python cli.py review --from 2026-04-01 --to 2026-04-10  # 指定时间段
python cli.py review 宁德时代                       # 复盘单只股票
```

**执行逻辑**：

1. 从报告库中拉取指定范围内的所有分析记录
2. 批量拉取每只股票分析日之后的实际走势（T+5/T+20 收益、最大回撤）
3. 逐条对比预测 vs 实际：
   - 方向是否正确
   - 幅度是否准确
   - 止损是否触发
   - 各维度哪些看对、哪些看走眼
4. 逐条进行**决策树节点回溯**：记录当时走的路径，分析哪个节点判断失误
5. 生成经验条目写入经验库
6. 输出汇总复盘报告 + 发邮件

**决策树节点回溯示例**：

```
宁德时代 2026-04-01 分析，预期差走的路径：
  有催化(是) → A类证据 → 30天内 → 未定价 → 单季超预期 → 78分
实际 T+20: -8.7%
诊断：Q4"市场定价程度"判断失误，分析时认为"未定价"，但股价近5日已涨12%
建议：Q4 增加子节点"近5日涨幅"作为辅助判断
```

### 通道 B：历史回测

嵌入夜间自进化引擎 22:00 轮次，定期批量回测。

**执行逻辑**：

1. 从报告库中取最近 N 次分析的完整数据包
2. 用当前决策树重新打分（纯代码，不调用 AI）
3. 对比 AI 评分 vs 决策树评分 vs 实际走势
4. 统计输出"决策树健康报告"：
   - 各维度预测力（与实际收益的相关系数）
   - 各节点区分度（高分组 vs 低分组收益差）
   - 整体胜率和盈亏比

**健康报告示例**：

```
【决策树健康报告】2026-04-10
样本：最近60次分析

维度预测力（与T+20收益相关系数）：
  预期差: r=0.42（强）
  资金面: r=0.35（中强）
  技术面: r=0.21（中）
  基本面: r=0.08（弱）→ 维持低权重

节点区分度预警：
  ⚠️ 资金面Q1"三者同向流入"区分度下降
     高分组(≥70)平均收益: +2.1%
     低分组(<70)平均收益: +1.8%
     差距仅0.3%，该节点需要细化

胜率统计：
  综合≥80分: 胜率62%, 平均收益+8.3%
  综合60-79: 胜率48%, 平均收益+1.2%
  综合<60分: 胜率31%, 平均收益-4.7%
```

### 通道 C：权重与决策树调节

**调节类型**：

| 类型 | 触发条件 | 示例 |
|------|---------|------|
| 权重微调 | 某维度相关系数连续30天变化>0.1 | 预期差提权至45% |
| 节点阈值调优 | 某节点区分度<1%持续20次 | "涨>20%算充分定价"改为15% |
| 新增分支 | 经验库同一tag出现≥5次 | 新增"催化-资金背离"子节点 |
| 市场状态适配 | 检测到市场环境切换 | 熊市中资金面权重提升 |

**审批流程（铁律：所有调节必须用户确认）**：

进化引擎生成调节建议后，通过邮件发送：

```
邮件标题：[LinDangAgent] 权重调节建议 - 2026-04-10

建议内容：
  预期差权重: 40% → 45%（+5%）
  基本面权重: 10% → 5%（-5%）

依据：
  近60次分析，预期差相关系数 r=0.42（最强）
  基本面相关系数 r=0.08（最弱且连续下降）

回测影响：
  采用新权重后历史胜率 52% → 55%（+3pp）
  高分组(≥80)收益 +8.3% → +9.1%

⏳ 回复"同意"生效，回复"否决"维持现状
```

变更生效后记录到 `weight_history.json`，可追溯完整变更历史。

## 经验检索机制

### 数据来源

经验库由通道 A（批量复盘）持续写入。

### 检索策略：三层匹配

| 优先级 | 匹配维度 | 命中方式 |
|--------|---------|---------|
| P0 | 同一只股票的历史分析 | 股票代码精确匹配 |
| P1 | 同行业 + 相似催化类型 | 行业标签 + 催化类型交集 |
| P2 | 相似技术形态 | 形态标签匹配 |

### 打分与排序

```
同股票: +10分
同行业: +5分
催化类型交集每个: +3分
形态标签交集每个: +3分
时间衰减: 30天内×1.0, 30-90天×0.7, 90天+×0.5
```

取 top 5 条，格式化为 `【历史镜鉴】` 文本块拼入数据包末尾。

### 实现方式

轻量文本检索（结构化标签匹配），不引入向量数据库。经验库规模在几百到几千条范围内，标签匹配足够精准且可解释性强。若未来超过 5000 条且标签覆盖不足，再考虑升级。
